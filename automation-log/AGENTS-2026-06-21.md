## หน้าที่แต่ละ agent
| agent | ไฟล์ | หน้าที่ |
|---|---|---|
| dispatcher | dispatcher.py | รับ orders → กระจายเข้าสภา → คิว |
| สภา 5 ชั้น | content_council.py | expert·comply·value·review·correct |
| comply gate | comply_gate.py | ตรวจ trigger/caveat หลัง CC + auto-fix + stamp |
| aggregator | executor.py | รวมคิว → work order → ส่ง CC |
| สะพาน CC | cc_bridge.py | send/collect cc-inbox/outbox + ping |
| execute สำรอง | cc_runner.py | ทำแทนเมื่อ CC ไม่ว่าง |
| รับผล→Cowork | report_to_cowork.py | cc-outbox → cowork-review |
| **6 แพลตฟอร์ม** | platform_agents.py | Pantip·FB·IG·TikTok·YouTube·Threads คอนเทนต์+แทคติก |
| **head creative** | head_creative.py | รวม 6 แพลตฟอร์ม → creative-brief → Cowork |
| **idea council** | idea_council.py | 4 มุมมอง เพิ่มความหลากหลาย/ยกระดับไอเดีย |
| **ปรึกษาภายนอก** | external_consult.py | Gemini3.5+Opus4.8 (เบราว์เซอร์) → รวมส่ง Cowork |
| **escalation** | ask_cowork.py | ไม่แน่ใจ → วนถาม Cowork |
| ตรวจลิงก์เว็บ | link_check.py | สแกน site/ ลิงก์ไฟล์เสีย + route → CC fix |
| art/graphic | art_agents.py | Art Director + Graphic + head of art → บรีฟปรับเว็บ → CC |
| รีวิวงาน CC | cc_review.py | รีวิวผล Claude Code → cowork-review → Cowork |

## คำสั่ง (cheat-sheet)
```
py pipeline\loop.py plan            # dispatcher→สภา→executor→CC inbox
py pipeline\loop.py collect         # ผล CC → Cowork review
py pipeline\head_creative.py        # 5 แพลตฟอร์ม → creative brief
py pipeline\idea_council.py "โจทย์"  # ปั่นไอเดียหลากมุม
py pipeline\platform_agents.py tiktok   # ทดสอบรายแพลตฟอร์ม
py pipeline\external_consult.py "คำถาม"  # บรีฟไปถาม Gemini+Opus (แล้ว collect <ts>)
py pipeline\ask_cowork.py           # ดูคำถามที่รอ Cowork ตอบ
```

## สิ่งที่เป็น "คน/Cowork" ไม่ใช่ cron เงียบ
- **โพสต์จริง** ทุกแพลตฟอร์ม = เจ้าของกดเอง (จาก draft ในคิว)
- **Gemini/Opus เบราว์เซอร์** = ต้อง login + Cowork ขับ Chrome ตอนเรียก (on-demand)
- **ตัดสินใจก้ำกึ่ง** = ask_cowork → Cowork ชี้ขาด
- **แตะเว็บ/ของจริง** = CC ใส่ "ธงต้องขออนุมัติ" รอเจ้าของ
