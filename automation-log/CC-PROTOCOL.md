# 📋 CC-PROTOCOL — คู่มือให้ Claude Code ทำงานในวงจร ngernduangold

Claude Code (CC) = ตัว execute แผน. รับงานจากกล่อง รันแล้วคืนผล — **มีขอบเขตชัดเจน**.

## ทำงานยังไง
1. อ่านงานใหม่ที่ `automation-log/cc-inbox/order-<ts>.md` (เก่าสุดก่อน)
2. ทำตาม "สิ่งที่ต้องทำ" ในออเดอร์ (ปกติ = แปลงคำตอบที่ผ่านสภา เป็นคอนเทนต์หลายแพลตฟอร์ม + วิเคราะห์ปรับปรุง)
3. เขียนผลลง `automation-log/cc-outbox/result-<ts เดียวกับออเดอร์>.md` (ฟอร์แมตด้านล่าง)
4. เสร็จแล้วหยุด — `report_to_cowork.py` จะเก็บผลไปให้ Cowork เอง

## ฟอร์แมตผล (cc-outbox/result-<ts>.md)
```
# CC RESULT <ts>
## เคส N: <ชื่อเคส>
**Threads:** <hook 1-2 บรรทัด + ชวน DM>
**Facebook:** <value 3-5 บรรทัด>
**TikTok:** <สคริปต์ ≤20 วิ>
**ปรับปรุง:** <1 ข้อเสนอ ยกระดับเนื้อหา/ฟันเนล>
## ธงต้องขออนุมัติ (ถ้ามี)
- <สิ่งที่อยากแก้ในเว็บ/โพสต์จริง — รอเจ้าของ OK>
```

## 🔒 ขอบเขต (ห้ามข้าม)
- ❌ ห้ามโพสต์ลงโซเชียลจริง ❌ ห้าม `git commit`/`push` ❌ ห้าม deploy/build เผยแพร่
- ❌ ห้ามใส่ลิงก์ในตัวโพสต์ (ลิงก์ = DM/bio เท่านั้น)
- ✅ ทำได้: ร่างคอนเทนต์, วิเคราะห์โค้ด/ฟันเนลแบบ read-only, เขียนไฟล์ผลใน cc-outbox
- งานที่ต้องแตะเว็บ/ของจริง → ใส่ใน "ธงต้องขออนุมัติ" ให้เจ้าของตัดสิน ไม่ทำเอง

## ใครสั่งให้ CC รัน
- เจ้าของเปิด Claude Code แล้วบอก "เคลียร์งานใน cc-inbox" — หรือ
- สั่งผ่าน Hermes/Telegram ให้ trigger CC (เจ้าของยืนยัน)
