# 🤝 HANDOFF — ระบบคิวคอนเทนต์อัตโนมัติ ngernduangold

> Cowork คุมงาน → dispatcher รับออเดอร์ → กระจายเข้าสภา 5 agent → คิว approve → แจ้ง Telegram
> **เจ้าของทำอย่างเดียว: เปิดคิว อ่าน กดโพสต์** (ลิงก์อยู่ DM/bio เท่านั้น)

---

## ภาพรวม (ใครทำอะไร)

```
Cowork (ตั้งออเดอร์)
   │   เขียนหัวข้อลง automation-log/orders.txt  (หรือปล่อยว่าง = ใช้ default 5 หัวข้อ)
   ▼
dispatcher.py  ── agent รับออเดอร์ + กระจายงาน
   │   วนทีละหัวข้อ → เรียกสภา content_council
   ▼
content_council.py ── สภา 5 agent (ใช้ free_llm: GLM→Qwen)
   1) expert    ร่างคำตอบ การเงิน/คปภ. value-first
   2) comply    ตรวจระเบียบ + คำต้องห้าม → PASS/FAIL
   3) value     เช็กว่ามีประโยชน์จริง + bridge DM แนบเนียน
   4) review    จับภาษาเพี้ยน/ข้อมูลมั่ว/ตัวเลขต้องเช็ก
   5) correct   แก้ให้ถูก สไตล์ Pantip ดอกเบี้ยเป็นช่วง+กำกับ "เช็กธนาคารอีกที"
   ▼
automation-log/council-<วันที่>.md  ── คิว "draft ตรวจแล้ว รอ approve"
   ▼
Hermes → Telegram เจ้าของ (chat 8431211539) แจ้ง "คิวพร้อม N รายการ"
   ▼
เจ้าของ: อ่านคิว → ก๊อปไปโพสต์เอง (Pantip/Threads/FB)  ← จุดเดียวที่ใช้คน
```

---

## ⏰ รันอัตโนมัติทุกเช้า (handoff แล้ว)

- **Windows Task:** `ngernduangold-daily-queue` — รันทุกวัน **07:30**
- รัน `pipeline/run_daily.cmd` → log ที่ `automation-log/dispatcher.log`
- ดูสถานะ: `schtasks /query /tn ngernduangold-daily-queue`
- เปลี่ยนเวลา: `schtasks /change /tn ngernduangold-daily-queue /st 06:00`
- สั่งรันเดี๋ยวนี้: `schtasks /run /tn ngernduangold-daily-queue`
- **ปิดถาวร:** `schtasks /delete /tn ngernduangold-daily-queue /f`

---

## 🎛️ สั่งงาน (3 วิธี)

1. **ปล่อยอัตโนมัติ** — ไม่ต้องทำอะไร 07:30 ได้คิว 5 หัวข้อมาตรฐานทุกวัน
2. **สั่งหัวข้อเอง** — แก้ `automation-log/orders.txt` (1 หัวข้อ/บรรทัด) แล้วรอเช้า หรือ `schtasks /run`
3. **ให้ Cowork สั่ง** — บอก Cowork "ปั่นคิววันนี้เรื่อง X, Y, Z" → Cowork เขียน orders.txt + รันให้

---

## 🔒 ขอบเขตความปลอดภัย (ออกแบบให้ handoff ได้อย่างสบายใจ)

ระบบนี้ **ผลิต draft ที่ตรวจแล้วเข้าคิวเท่านั้น** — โดยตั้งใจให้ทำไม่ได้:
- ❌ โพสต์เอง (ทุกคำตอบรอคนกดโพสต์)
- ❌ commit / push / deploy โค้ด
- ❌ ใส่ลิงก์ในคำตอบ (ลิงก์อยู่ DM/bio เท่านั้น — กันสแปม + ระเบียบ)
- ❌ ใช้โมเดลเสียเงิน (เรียง GLM ฟรีก่อน, rotate เมื่อเต็ม)

→ ความเสี่ยงสูงสุด = draft ในคิวไม่สวย → เจ้าของไม่กดโพสต์ จบ ไม่มีผลออกสู่สาธารณะ

## 🔁 โมเดลฟรี (free_llm.py)
